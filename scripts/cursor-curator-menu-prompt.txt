# Budapest Night — Menu & Eat/Drink curator (copy below into a new chat)

---
You enrich **existing venues** with published food & drink menus so `/eat-drink` and themed tours (pálinka, foodie, coffee) work.

Use tools (web search, fetch, terminal, **image read/vision**). Prefer **batch patches** (5–15 venues per run) when safe.

## Menu only as PNG/JPG on the site (required workflow)
Many Budapest venues publish the drink/food list as **menu board images** (WordPress uploads), not HTML tables. **Do not skip** these venues.

**Menu images ≠ listing hero:** OCR menu boards for `menu` sections only. The venue’s public card `image` must still be a proper venue photo (interior/exterior), not a cropped menu PNG — see `src/lib/curator/imageIngestRules.ts`.

1. Open the venue’s menu page and collect every official menu image URL (e.g. `…/wp-content/uploads/…/MENU_….png`).
2. **Download** each image under `scripts/menu-sources/<venue-slug>/` (keep filenames; note URLs in the payload `sourceUrls`).
3. **Read** each downloaded image (agent vision or `tesseract` in terminal) and transcribe names, sizes, and **HUF/EUR** prices as printed.
4. Build the ingest patch from that transcription; use `price.source: "published"` when taken from the image.
5. List the **image URL(s)** in `menu.sourceUrls` plus the menu landing page. Only use `estimated` + `missingOrUncertain` for lines you could not read on the image.

## Goal
Increase `providersWithMenu` and menu item count on production. Each patched venue must have real `menu.sections[].items[]` with `kind`, `tags`, prices when printed, and **translated menu copy** for every locale.

**Menu URLs must be on the venue’s live `website` domain** (e.g. `/eteleink`, `/italaink` on `kiosk-budapest.hu`). If `website` is dead or a different hostname, fix the provider contact fields first — do not transcribe menus from a stale domain.

Verify after ingest:
- `GET https://budapest-night.vercel.app/api/public/menu-items?locale=it&tag=palinka&limit=20` — item **names** and section context in Italian
- `GET https://budapest-night.vercel.app/api/public/tours/palinka?locale=it` → not `not_enough_venues`

## Scarcity (menus)
Prioritize venues that unlock tours and tag coverage:

1. **No menu body** — providers missing `menu.sections` (ignore orphan `menuTags` without items).
2. **Tour tags** — need more venues with items tagged `palinka`, (`hungarian`|`goulash`|`street-food`), or (`coffee`|`specialty-coffee`).
3. **Category** — Restaurants & Cafés first for foodie/coffee; Parties/Venues for palinka.
4. **District** — spread patches across boroughs.

## Menu translations (required)
**Menu content must be translated** — the Eat & Drink UI is localized (`/it/eat-drink`, `/hu/eat-drink`, etc.). English-only dish names will show in every language.

- Root **English** on each item: `name`, optional `description`; on each section: `title`.
- **`locales`** on **every** item and section with keys **`hu`, `es`, `it`, `he`, `ar`** (no `en` inside `locales`).
- Each locale block:
  - Item: `{ "name": "…", "description": "…" }` — `name` required; translate as guests read on a menu in that language.
  - Section: `{ "title": "…" }` — e.g. `"Drinks"` → `"Bevande"` (it), `"Italok"` (hu).
- Keep brand names, pálinka, sizes (4 cl), and dish names that are conventionally untranslated when the source menu does so.
- **Never** put `tags`, `price`, `kind`, or `id` inside `locales` — root only.
- Prices stay in `price` on the root item (HUF/EUR as printed).

## Patch shape (preferred)
Do **not** re-upsert whole venue unless needed. Patch an **existing** `prov-*` id:

    {
      "resource": "provider",
      "action": "patch",
      "id": "prov-existing-id",
      "patch": {
        "menu": {
          "menuUrl": "https://...",
          "sourceUrls": ["https://..."],
          "lastVerifiedAt": "2026-05-16",
          "sections": [
            {
              "id": "drinks",
              "title": "Drinks",
              "kind": "drink",
              "locales": {
                "hu": { "title": "Italok" },

                  "id": "house-palinka-plum",
                  "kind": "drink",
                  "name": "House plum pálinka (4 cl)",
                  "tags": ["palinka"],
                  "locales": {
                    "hu": { "name": "Házi szilvapálinka (4 cl)" },
                    "es": { "name": "Pálinka casera de ciruela (4 cl)" },

## Rules (blocking)
- **Do not** send `menu.venueLink` or `menuTags` — computed on ingest from the provider row.
- Every menu patch must target a real `prov-*` id that exists (or is upserted earlier in the same payload).
- **Every** section and item must include full `locales` (hu, es, it, he, ar) — ingest rejects otherwise.
- Tags only from: `palinka`, `wine`, `beer`, `craft-beer`, `cocktail`, `coffee`, `specialty-coffee`, `goulash`, `hungarian`, `street-food`, `dessert`, `vegan`, `vegetarian`, `ruin-bar`, `rooftop`, `danube-view`
- **HUF** vs **EUR** must match the menu (1490 Ft ≠ €1490).
- 3–12 representative items per venue; include food **and** drink sections when the source has both.
- Only `published` prices unless noted in `missingOrUncertain`.

## Output file
`scripts/ingest-payloads/cursor-curated-menu-batch-<date>.json`

## Ingest
`npm run ingest:listing` validates **provider patch** menus (sections, items, and `locales` on every item/section) before POST. Fix all dry-run errors before `--force`.

```bash
npm run ingest:listing -- --dry-run scripts/ingest-payloads/cursor-curated-menu-batch-<date>.json
npm run ingest:listing -- --force scripts/ingest-payloads/cursor-curated-menu-batch-<date>.json
```

Legacy payloads without menu translations may pass only with `--skip-locale-check` (provider profile only — **not** a substitute for menu `locales`).

After deploy, optional local repair:
`npm run db:recompute-menu-tags`
