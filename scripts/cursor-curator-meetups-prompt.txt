# Budapest Night — Culture circle curator (copy below into a new chat)

---
You are my **culture circle** curator for Budapest Night. Use tools (web search, fetch, terminal). Ingest **one** meetup group (`resource: "meetupGroup"`), not a venue upsert alone and not a single concert.

## Goal
Add one Budapest **culture circle** with production ingest `HTTP 200`, `ok: true`, a **unique** ImgBB `coverImageUrl`, and optional `venueIds` / `eventIds` when the circle hosts at real catalog rows.

## Read first (repo)
- **Location rules (blocking):** `scripts/cursor-curator-location-rules.txt`
- **Image rules (blocking):** `src/lib/curator/imageIngestRules.ts` — especially `getMeetupCoverImageIngestRulesForPrompt()`
- Enums: `src/lib/curator/constants.ts` (meetup group types), neighborhoods: `src/data/locations.ts`
- Main curator: `scripts/cursor-curator-prompt.txt` (meetup JSON shape)
- Repair: `npm run db:patch-venue-meetup-images`

## Live catalog (mandatory)
```bash
curl -s https://budapest-night.vercel.app/api/public/meetup-groups
curl -s https://budapest-night.vercel.app/api/public/providers
```

Dedupe by `id` and name. **Image audit:** list every `coverImageUrl`; your new URL must not match any existing row.

## Scarcity (meetups only)
Rank **groupType** → **district** → **neighborhood** → **cadence** (fewest first). Document counts in `notes`.

## Cover image (blocking — production lesson)

Culture circles are **community / district** groups, not concerts.

| Wrong | Why |
|-------|-----|
| Same `coverImageUrl` on 10+ groups in one batch | Grid shows identical Johannes Oerding / concert art on every card |
| Concert poster from a ticket page | Belongs on `resource: "event"` only |
| Copying another meetup’s URL from the catalog | Broken Discover UI |
| Reusing hash `cde3b78d5c56` | Known bad bulk upload — **forbidden** |

**Correct:**
1. Choose a **district-appropriate** photo (linked venue site, circle `website` `og:image`, or Wikimedia scene of that neighborhood — verify HTTP 200).
2. Download → upload **once per group** → unique `https://i.ibb.co/...`
3. Confirm only **one** meetup row uses that URL in `GET /api/public/meetup-groups`

Do **not** reuse placeholder files that share the same bytes across `scripts/imgbb-asset-sources/providers/cov-cov-meet-*.jpg` without checking MD5.

## Links (optional)
- `venueIds`: existing `prov-*` hosts — upsert providers **before** the meetup in `operations[]`
- `eventIds`: existing `event-*` — upsert events **before** the meetup
- Do **not** send `venueLinks` / `eventLinks` (computed on ingest)

## Ingest
```bash
npm run ingest:listing -- --dry-run scripts/ingest-payloads/cursor-curated-meetup-<slug>.json
npm run ingest:listing -- scripts/ingest-payloads/cursor-curated-meetup-<slug>.json
```

## Final reply
- Name, id, groupType, borough, neighborhood
- **Cover:** original source URL + final ImgBB URL
- **Uniqueness:** confirm no other meetup shares this `coverImageUrl`
- Scarcity counts before/after
- Linked `venueIds` / `eventIds` if any
