## Lessons from recent production fixes (must follow)

### 1. Event images ≠ venue images
Event cards must show **artist / show promo art**, not the host venue’s listing photo.

| Wrong | Why |
|-------|-----|
| Copying `provider.image` onto the event | Users see duplicate venue photos on every card |
| Budapest Park generic “leave reality outside” graphic for all Park shows | Same image on 6+ events — fixed by per-show art |
| MVM Dome **building exterior** for Sting | Arena photo ≠ tour artwork; use promoter/tour image |
| Reusing one ImgBB URL on multiple events | Catalog looks broken in Featured + list views |

**Correct:** scrape **per-show** art from the official event or ticket page, upload to ImgBB, assign **one unique URL per event**.

### 2. How to find the right image URL
1. Open the **event/ticket page** (not only the venue homepage).
2. Read `og:image` / `twitter:image` from HTML, or listing-specific assets:
   - **budapestpark.hu** — prefer `budapestpark.cdn-netpositive.com/event_images/...` or `/uploads/<hash>.{jpg,png}` from the **ticket/event URL**; ignore site-wide `budapestpark-og-2021.jpg`.
   - **mupa.hu** — hero under `/storage/media/.../conversions/...` for that program.
   - **livenation.hu** — tour artwork from `dynamicmedia.livenationinternational.com/...` when `og:image` is missing; not the arena stock photo.
   - **barbanegra.hu** — `/uploads/<show-slug>.jpg` on the show page.
   - **akvariumklub.hu** — event page `og:image` (band photo), not site chrome icons.
   - **durerkert.com** — Minio/CDN poster for that event.
3. `curl` or fetch the image URL; confirm it is a real photo/poster (not favicon/logo).
4. Save under `scripts/imgbb-asset-sources/events/<slug>.{jpg,png,webp}`.
5. Upload to ImgBB **before** ingest:
   ```bash
   # Option A: project upload API (needs INGEST_API_KEY)
   curl -s -X POST https://budapest-night.vercel.app/api/ingest/upload \
     -H "Authorization: Bearer $INGEST_API_KEY" \
     -F "file=@scripts/imgbb-asset-sources/events/artist-venue.jpg"

   # Option B: IMGBB_API_KEY in .env.local (see scripts/patch-listing-images.ts)
   ```
6. Put the returned `https://i.ibb.co/...` URL in `document.image`.
7. Cross-check `GET /api/public/events` — your new URL must not match any existing event’s `image`.

If you already ingested with a bad image: patch only:
```json
{ "resource": "event", "action": "patch", "id": "event-...", "patch": { "image": "https://i.ibb.co/..." } }
```
Or extend `scripts/patch-event-images.cjs` and run `npm run db:patch-event-images`.

### 3. Schedule when only gates are published
Many open-air pages list **gate opening 18:00** but not stage time. Unless the source gives an explicit show start:
- `doorsOpenAt`: published gate time (e.g. `2026-07-07T18:00:00+02:00`)
- `startsAt`: **20:00** same day (document in `missingOrUncertain`)
- `endsAt`: ~23:00–23:30 for concerts; **long opera** use published end (e.g. Parsifal 16:00–22:20)
- `timezone`: `"Europe/Budapest"`

### 4. Venue IDs and districts (common pitfalls)
| Host | Typical `venueIds` | Event `borough` (from host) |
|------|-------------------|-----------------------------|
| Budapest Park | `prov-budapest-park-obuda` | **Ferencváros** / Fábián Juli (1095 — not Óbuda Island) |
| MVM Dome / Live Nation arena | `prov-mvm-dome-ujbuda` | **Terézváros** / Stefánia út (1143) |
| Müpa Bartók Hall | `prov-cov-bartok-moricz` | Ferencváros / Műegyetem |
| A38 Ship | `prov-a38-ferencvaros` | Ferencváros / Petőfi Bridge |
| Barba Negra | `prov-barba-negra-ujbuda` | **Újbuda** — show may be Red Stage (Szállító u.); follow **show page** address |
| Akvárium | `prov-akvarium-belvaros` | Belváros |

Provider id suffixes like `-obuda` are legacy slugs; trust the provider’s **address + borough** in the catalog, not the id string.

### 5. Tickets
- Forint shops → `currency: "HUF"`, integer amounts (e.g. `17999` = 17,999 Ft).
- Paid show with unknown tiers → `entryFees: []` + explain in `missingOrUncertain` — **never** `FREE` / `amount: 0`.
- Dynamic pricing (Müpa) → one `from` tier + `notes` about range; do not invent upper tiers.
- Admin/handling fees → `notes` on the tier, not added to `amount`.

## Scarcity (events catalog)
Measure the **events** collection separately from providers. Rank gaps:

1. **Upcoming window** — prefer shows in the next 3–12 months with `status: "scheduled"`.
2. **District (`borough`)** on the event — fewest upcoming events per district.
3. **Month** — sparse months in the calendar.
4. **Activity** — e.g. Electronic, Live Music, Festival (root `activityTypes`, English).
5. **Tag gaps** — boroughs with many venues but zero upcoming timed events.

Pick real official listings that fill the scarcest slice you can source credibly.

## Hard requirements (checklist)
- [ ] Official promoter or venue ticket page; cite in `Sources:` lines (EN + all locales).
- [ ] `startsAt` / `endsAt` / optional `doorsOpenAt` — ISO 8601 with offset (`+02:00`); `timezone: "Europe/Budapest"`.
- [ ] `venueIds` → existing `prov-*` (primary host **first**). Upsert venues **before** events in `operations[]`.
- [ ] Event `borough` / `neighborhood` match primary host (ingest overwrites from host — still set correctly).
- [ ] `entryFees`: real tiers in HUF/EUR; `[]` only when truly unknown.
- [ ] **`image`**: unique ImgBB URL from **this show’s** promo art (see workflow above).
- [ ] `locales`: `hu`, `es`, `it`, `he`, `ar` — each with `title`, `shortDescription`, `longDescription`, `slug`.
- [ ] Root copy in **English**; no `locales.en`.
- [ ] `website` = event page; `bookingUrl` = direct ticket URL when available.
