# Budapest Night — Location rules (blocking)

Full venue checklist: docs/venue-records.md

Copy this entire file into any curator chat that sets `borough`, `neighborhood`, or `address`.
Registry version: see `version` in `src/data/budapest-location-registry.json`.

---

## Source of truth (only these)

| File | Purpose |
|------|---------|
| `src/data/budapest-location-registry.json` | Postal → app borough, verified landmarks, street overrides, forbidden pairings |
| `src/data/locations.ts` | Allowed `neighborhood` strings per `borough` |
| Official venue website / contact page | Street address + postal code for the payload |

**Forbidden as location sources:** marketing prose, `prov-*` id suffix, coverage `gapKey`, “Millennium quarter”, “Infopark” in copy, memory, other listings.

---

## Address format (required)

```
{postal} Budapest, {street}, Hungary
```

Example: `1124 Budapest, Csörsz utca 18, Hungary`

---

## Resolution order (automated — ingest enforces)

1. **landmark** — `landmarks[]` entry matching `document.id`
2. **landmark** — same normalized address as a registered landmark
3. **addressOverride** — street pattern in registry (e.g. `csörsz` → Buda / MOM Park)
4. **postal** — `postalToAppBorough[postal].appBorough` + neighborhood from allow-list

If postal is missing from registry → **dry-run fails**. Add the postal row (with `kerulet`, `keruletName`, `appBorough`) before ingest.

## Checklist before ingest

- [ ] Postal code copied from official site (not invented)
- [ ] `borough` + `neighborhood` match registry resolution
- [ ] `neighborhood` is exact allow-list string from `src/data/locations.ts`
- [ ] No forbidden copy (Infopark on Csörsz, Óbuda Island on Budapest Park, etc.)
- [ ] `npm run ingest:listing -- --dry-run <payload.json>` passes location errors
